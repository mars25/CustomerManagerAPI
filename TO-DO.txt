PASO 1 — Crear la solución en .NET

En tu máquina, abre una terminal dentro de la carpeta donde trabajarás y ejecuta:

1) Crear solución
dotnet new sln -n CustomerManagerAPI

2) Crear proyecto Web API
dotnet new webapi -n CustomerManagerAPI.Api

3) Agregar proyecto a la solución
dotnet sln add CustomerManagerAPI.Api


Esto crea la API y la conecta con la solución.

✅ PASO 2 — Crea la estructura de carpetas dentro del proyecto

Dentro de CustomerManagerAPI.Api, crea carpetas así:

/Controllers
/DTOs
/Entities
/Services
/Repositories
/Mapping
/Configurations


Esto muestra a clientes buena arquitectura.

✅ PASO 3 — Crea modelo inicial (Customer Entity)

Crea el archivo:
Entities/Customer.cs

public class Customer
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Email { get; set; }
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;
}

✅ PASO 5 — Instalar Entity Framework Core + SQL Server

En terminal, dentro del proyecto API:

dotnet add package Microsoft.EntityFrameworkCore 6.0.0
dotnet add package Microsoft.EntityFrameworkCore.SqlServer 6.0.0
dotnet add package Microsoft.EntityFrameworkCore.Tools
dotnet add package AutoMapper
dotnet add package AutoMapper.Extensions.Microsoft.DependencyInjection

✅ PASO 6 — Crear el DbContext

Crea el archivo:

Configurations/AppDbContext.cs

using Microsoft.EntityFrameworkCore;
using CustomerManagerAPI.Api.Entities;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }

    public DbSet<Customer> Customers { get; set; }
}

✅ PASO 7 — Registrar el DbContext en Program.cs

En Program.cs, agrega:

builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));


Y en appsettings.json agrega:

"ConnectionStrings": {
  "DefaultConnection": "Server=localhost;Database=CustomerManager;Trusted_Connection=True;TrustServerCertificate=True;"
}
